Tôi đã kiểm tra và thấy lỗi chính là do thiếu package `swagger-jsdoc` và `swagger-ui-express` trong thư mục node_modules khi bạn pull code từ Git về.
Tôi đã chạy lệnh `npm install` để cài đặt các package này và server hiện đã khởi động thành công (tại http://localhost:3000).

Về Database, sau khi kiểm tra kỹ các thay đổi trong code mới nhất (`server.js`) bao gồm các query:
- Bảng `tasks` (cột `is_new`, `description`, `assigned_to`, v.v.)
- Bảng `work_logs` (cột `edit_reason`, `feedback`, `actual_grade`, `attachment_url`, v.v.)
- Bảng `users` (cột `accumulated_points`)
- Bảng `kpi_settings`

Tất cả các cột và bảng này ĐÃ TỒN TẠI VÀ ĐỒNG BỘ hoàn toàn với code hiện tại. 
Do đó, tôi KHÔNG CẦN thay đổi hay sửa gì thêm ở dưới Database. Code hiện tại đã chạy ổn định với cấu trúc Database sẵn có.

Nếu bạn gặp lỗi Database ở máy 2, hãy đảm bảo rằng bạn đã chạy tất cả các file migrate (`alter_db.js`, `alter_db_v2.js`, `alter_db_v3.js`) trên máy 2 nhé!

**CẬP NHẬT MỚI (Ngày 11/05/2026):**
Đã bổ sung thêm 3 cột mới vào bảng `tasks` để phục vụ tính năng Giao Việc (Push Task) và Cảnh báo quá hạn:
1. `deadline` (DATETIME): Thời hạn hoàn thành task.
2. `attachment_url` (VARCHAR 255): Đường dẫn lưu trữ file đính kèm.
3. `attachment_name` (VARCHAR 255): Tên gốc của file đính kèm.

Lưu ý: Backend đã tự động chạy lệnh `ALTER TABLE` khi khởi động server, nên khi bạn chuyển sang máy khác, chỉ cần pull code về và chạy lại `node server.js` là Database sẽ tự động đồng bộ (bổ sung các cột trên).
